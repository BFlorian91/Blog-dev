En tant que débutant en programmation on nous inculque forcément les **_if, else statement_**. C'est l'essence même de la programmation, fait ça si truc est égale à, sinon fait ça.

Le soucis, c'est qu'ils sont souvent utilisés à outrance et sans réèl utilité comme dans cet exemple ci-desous:

<h2 style="color: #ff5e78"># Bad Practice</h2>

#### 1-  **If, Else if, Else Statement**

```js
// if else statement #dart
  int statusCode = 404;

  if (statusCode == 404) {
    print('OOpsi tu es perdu !!');
  } else if (statusCode == 401) {
    print('Hum.. t\'es qui toi? 🤭');
  } else if (statusCode == 500 || statusCode == 503) {
    print('Sorry, Roger c\'est pris les pieds dans les cables ...');
  } else {
    print('Bon surf buddy 😏');
  }

```
#### 2- **If, Else**

```js
// Code status très douteux 😂 #dart
  int status(int code) {
    if (code == 404) {
      print('OOpsi tu es perdu');
      return 1;
    else
      print('Ça c\'est peut être bien passé');
      return 0;
    }
  }
```

Je sais pas ce que vous en pensez, mais tout ça peut vite devenir très fouillit, et on risque d'annéantir toute volonté de compréhension d'un collègue ou même d'un moi du future !

<h2 style="color: #18C79A"># Good Practice</h2>

#### -1 **Switch Statement**
```js
// Switch statement #dart
  int statusCode = 404;

  switch (statusCode) {
    case 404:
      print('OOpsi tu es perdu !!');
      break;
    case 401:
      print('Hum as tu plus de 18 ans 🤭');
      break;
    case 500 || 503:
      print('Sorry, Roger c\'est pris les pieds dans les cables ...');
      break;
    default:
      print('Bon surf buddy 😏');
      break;
}
```

#### -2 **If statement**

```js

int status(int code) {
  if (code == 404) {
    print('OOpsi tu es perdu');
    return 1;
  }
  print('Ça c\'est peut être bien passé');
  return 0;
}
```

Pour le dernier exemple on voit que j'ai supprimé le ***else***, ça parraît futile, mais en réalité vous pouvez toujours faire ça, donc à quoi bon surchargé votre code de charactères inutiles ?
